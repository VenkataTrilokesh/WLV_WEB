01 / k-WL Test

The Weisfeiler–Lehman
Graph Isomorphism Test

A powerful combinatorial algorithm that iteratively refines node colors based on neighborhood structure to detect graph non-isomorphism.

Color Refinement O(n² log n) k-dimensional
Core Concept

What is the WL Test?

The Weisfeiler–Lehman algorithm assigns colors to nodes by repeatedly hashing each node's color with its neighbors' colors. Two graphs that produce different color histograms at any iteration are definitely not isomorphic.

🎨
Color Refinement
Each node starts with a color based on its degree. Iteratively, each node's color is updated by hashing it with the multiset of neighbor colors.
🔁
Convergence
The process terminates when the color partition stabilizes — no new color classes are created. This stable partition is the graph's WL certificate.
⚖️
Comparison
Two graphs are compared by their final color histograms. A mismatch at any iteration proves non-isomorphism. A match is necessary but not sufficient.
Algorithm Steps

How It Works

c⁰(v) = deg(v)
cᵢ⁺¹(v) = HASH( cᵢ(v), sort({ cᵢ(u) : u ∈ N(v) }) )
1
Initialize Colors
Assign each node a color based on its degree (number of neighbors). Nodes with the same degree start with the same color.
2
Aggregate Neighborhoods
For each node, collect the sorted multiset of its neighbors' current colors. This captures the local structural context.
3
Hash & Relabel
Compute a new color for each node by hashing (own color, neighbor color multiset). Equal tuples always map to the same new color.
4
Check Convergence
If the color partition hasn't changed, the algorithm terminates. Otherwise, repeat from step 2.
5
Compare Certificates
The final color histogram is the WL certificate. Different certificates → graphs are not isomorphic. Same → inconclusive.
k-Dimensional Extension

Beyond 1-WL: The k Parameter

Higher k means the algorithm considers k-tuples of nodes, greatly increasing discriminating power at the cost of computational complexity.

kConsidersComplexityPower
k = 1Individual nodes + neighborsO(n²)Standard WL — detects most non-isomorphic pairs
k = 2Pairs of nodesO(n³)More powerful; detects some k=1 failures
k = 3Triples of nodesO(n⁴)Captures triangle and 3-clique structure
k ≥ 4k-tuplesO(nᵏ⁺¹)Highly discriminating; impractical for large n
